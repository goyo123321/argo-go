DNS Configuration Explained

Understanding DNS-over-HTTPS (DoH) and Traditional DNS Protocols

DNS Configuration Example

This is a typical DNS configuration found in network tools like Surge, Shadowrocket, or Clash:

dns: { servers: [ "https+local://8.8.8.8/dns-query", "https+local://1.1.1.1/dns-query", "8.8.8.8", "1.1.1.1" ] }

This configuration provides a balanced approach with both encrypted DNS-over-HTTPS and traditional DNS for reliability and performance.

Protocol Breakdown

DNS-over-HTTPS (DoH)

Format: https+local://8.8.8.8/dns-query

  • https:// - Uses DNS-over-HTTPS protocol
  • +local - Direct connection (bypasses proxy)
  • 8.8.8.8 - Google's DNS server
  • /dns-query - Standard DoH endpoint

Traditional DNS

Format: 8.8.8.8

  • Uses standard UDP protocol
  • No encryption (vulnerable to snooping)
  • Generally faster response times
  • Widely compatible

Protocol Comparison

Protocol Type Security Speed Anti-interference Use Cases
DoH (+local) ✓ High (Encrypted) ~ Medium ✓ Strong Privacy protection, anti-hijacking
Traditional DNS ✗ No encryption ✓ Fast ✗ Vulnerable to hijacking Fast resolution, compatibility

How It Works

1

Priority Order

DNS servers are tried in the order listed in the configuration

2

Fallback Mechanism

If the first DoH server fails, the system automatically tries the next one

3

Hybrid Approach

Combines encrypted DoH for security with traditional DNS for speed and reliability

Practical Application

This DNS configuration is commonly used in:

  • Network proxy tools (Surge, Shadowrocket, Clash)
  • VPN applications with custom DNS settings
  • Privacy-focused browsing setups
  • Regions with DNS censorship or interference

The configuration provides:

  • Privacy protection through encrypted DoH queries
  • Reliability with multiple server redundancy
  • Flexibility with direct connections ensuring DNS resolution isn't affected by proxy settings
  • Compatibility with traditional DNS as a fallback
